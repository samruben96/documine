## Lessons Learned

### 1. Quick Polish Sprints Work
Between major feature epics, a focused polish sprint (4-10 stories, 1-2 days) is valuable:
- Addresses accumulated UX friction
- Provides sense of progress
- Low cognitive load
- Sets up a cleaner foundation for next epic

### 2. Skeleton Loaders Are High-ROI
Adding skeleton loaders is minimal effort with outsized UX impact. Should be standard practice for any data-fetching component.

### 3. YOLO Mode for Lightweight Retros
Quick polish sprints don't need full retrospective ceremony. The `/retrospective` workflow's YOLO mode is perfect for:
- Single-day epics
- Polish/bugfix sprints
- Clear-cut deliveries with no issues

## Key Issue Identified: `epic-yolo` Workflow Not Exposed

### Problem
Sam identified that the `*epic-yolo` workflow command wasn't being recognized. Investigation revealed:

1. **Workflow EXISTS:** `.bmad/bmm/workflows/4-implementation/epic-yolo/workflow.yaml` ✅
2. **Instructions EXIST:** `.bmad/bmm/workflows/4-implementation/epic-yolo/instructions.md` ✅
3. **Manifest REGISTERED:** Entry in `.bmad/_cfg/workflow-manifest.csv` ✅
4. **Slash Command MISSING:** `/bmad:bmm:workflows:epic-yolo` NOT in Claude Code Available Commands ❌

### Root Cause
The BMAD system correctly has the workflow defined and registered, but Claude Code's command integration is missing the corresponding entry. Other Phase 4 implementation workflows (dev-story, code-review, etc.) ARE registered as slash commands, but epic-yolo was either:
- Added after installation
- Missed during command registration
- Not synced properly

### `epic-yolo` Workflow Purpose
Rapid epic execution with smart status handling:
- **backlog** → runs `create-story` → **drafted**
- **drafted** → runs `story-context` → **ready-for-dev**
- **ready-for-dev** → runs `dev-story` → **review**
- **review** → runs `code-review` → **done**

Automatically processes ALL stories in an epic through their full lifecycle without pauses.

### Action Required
The BMAD system needs to expose `/bmad:bmm:workflows:epic-yolo` as a slash command. This may require:
1. Re-running BMAD installer to sync commands
2. Manual command registration
3. Bug report to BMAD maintainers

### Workaround
Use BMad Master agent and the `*epic-yolo [epic-id]` asterisk command instead of the slash command.
